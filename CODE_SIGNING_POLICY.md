# 코드 서명 정책

이 문서는 `ebook_viewer.exe` 배포 시 코드 서명과 릴리스 관리 기준을 정의합니다.

## 1. 대상 산출물

- `ebook_viewer.exe`

## 2. 릴리스 기준

- 모든 공개 배포 파일은 공개 저장소의 태그 기준으로 생성합니다.
- 릴리스 노트에는 버전, 변경 내용, 대상 파일을 적습니다.
- 릴리스 파일은 가능하면 SHA256 해시를 함께 제공합니다.
- 버전 문자열은 소스와 배포 파일 메타데이터에서 일치해야 합니다.

## 3. 서명 기준

- 공개 배포 파일은 코드 서명 후 배포합니다.
- 서명 후 파일을 다시 수정하지 않습니다.
- 서명된 파일의 이름, 버전, 해시를 릴리스 정보와 맞춥니다.

## 4. 빌드 기준

- 빌드는 문서화된 명령으로 재현 가능해야 합니다.
- 필요한 도구 버전은 README에 명시합니다.
- 빌드 산출물은 공개 저장소와 태그 기준으로 추적 가능해야 합니다.

## 5. 저장소 기준

- 저장소는 공개 상태를 유지합니다.
- 루트에 `LICENSE` 파일을 둡니다.
- README에 프로젝트 목적, 빌드 방법, 배포 파일 설명을 적습니다.
- 태그와 릴리스 정책을 유지합니다.

## 6. 보안 기준

- 릴리스 권한은 관리된 계정으로만 사용합니다.
- 가능하면 릴리스 계정에 MFA를 적용합니다.
- 코드 서명 관련 자격 증명 또는 서비스 접근 권한은 최소 인원만 사용합니다.

## 7. 오탐 대응 기준

- 서명 후에도 보안 제품 오탐이 발생하면 해시 기준으로 오탐 신고를 진행합니다.
- 신고 시 제품명, 버전, 배포 URL, SHA256, 공개 저장소 URL을 함께 제공합니다.
- 릴리스마다 동일한 제품명과 버전 체계를 유지합니다.

## 8. 문서 유지 기준

- 빌드 절차가 바뀌면 README와 이 문서를 함께 수정합니다.
- 릴리스 정책이 바뀌면 태그 규칙과 버전 규칙을 함께 갱신합니다.
- 배포 구조가 바뀌면 서명 대상 산출물 목록부터 다시 검토합니다.
