# 버전 및 태그 규칙

이 문서는 `ebook-viewer-oss` 저장소의 버전 번호와 Git 태그 규칙을 정리합니다.

## 1. 기본 규칙

- 버전 형식은 `Major.Minor.Patch`를 사용합니다.
- Git 태그는 `v1.0.0` 형식을 사용합니다.
- `package.json`의 `version`과 공개 릴리스 버전은 일치시킵니다.
- EXE 메타데이터 버전과 릴리스 태그도 가능한 한 맞춥니다.

## 2. 올리는 기준

### Patch

- 문서 수정
- 오타 수정
- 안정성 개선
- 작은 UI 수정

예:

- `v1.0.1`

### Minor

- 기능 추가
- 설정 옵션 추가
- 빌드 구성 개선

예:

- `v1.1.0`

### Major

- 앱 구조 변경
- 호환성 영향이 큰 변경
- 배포 방식 변경

예:

- `v2.0.0`

## 3. 추천 첫 릴리스

- Git 태그: `v1.0.0`
- `package.json` 버전: `1.0.0`
- 파일 버전: `1.0.0.0` 또는 이에 맞는 값

## 4. 릴리스 체크

- `package.json` 버전 확인
- README 최신화 완료
- `LICENSE` 확인
- 빌드 산출물 생성 완료
- SHA256 생성 완료
- Git 태그 생성 완료

## 5. 태그 예시

```powershell
git tag v1.0.0
git push origin v1.0.0
```
